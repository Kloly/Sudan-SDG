# نبض السوق — متتبّع سعر الصرف الموازي في السودان

موقع بسيط يجمع تلقائيًا سعر الدولار وعملات أخرى في **السوق الموازي بالسودان** كل ساعة، ويعرضها مع رسم بياني للتاريخ. لا يحتاج سيرفر دائم — يعمل بالكامل عبر GitHub (مجاني).

## كيف يعمل

1. `scraper.py` يجلب الأسعار من صفحة العملات في [alsoug.com](https://www.alsoug.com/en/currency) (عمود "Unofficial" = السوق الموازي).
2. يضيف لقطة جديدة بالتاريخ والوقت إلى `data/history.json`.
3. `.github/workflows/update-rates.yml` يشغّل هذا السكريبت تلقائيًا **كل ساعة** عبر GitHub Actions، ويحفظ (commit) التحديث في المستودع.
4. `index.html` صفحة ثابتة تقرأ `data/history.json` وتعرض آخر الأسعار + رسم بياني تفاعلي.

## خطوات النشر (10 دقائق)

1. أنشئ مستودع جديد على GitHub وارفع له كل هذه الملفات كما هي (بنفس بنية المجلدات).
2. من **Settings → Actions → General → Workflow permissions** اختر **Read and write permissions** (ضروري حتى يستطيع الووركفلو حفظ تحديثات `history.json`).
3. من **Settings → Pages** فعّل النشر من فرع `main` والمجلد `/ (root)`.
4. بعد دقيقة أو دقيقتين سيظهر رابط الموقع (شكله: `https://USERNAME.github.io/REPO-NAME/`).
5. لتجربة الجمع فورًا بدون الانتظار للساعة القادمة: تبويب **Actions** → اختر الووركفلو → **Run workflow**.

## ملاحظات مهمة

- **هشاشة الجمع التلقائي**: لو غيّر موقع alsoug.com تصميم صفحته، قد يفشل السكريبت. ستجد رسالة الخطأ في تبويب Actions. الحل: افتح الصفحة وعدّل `NAME_MAP` أو منطق التحليل في `scraper.py` بما يطابق البنية الجديدة.
- **مصدر واحد حاليًا**: لزيادة الدقة يمكن إضافة مصدر ثانٍ (مثل صفحات الأسعار في مواقع إخبارية سودانية) ومتوسطة القراءتين — أخبرني إن رغبت أضيف هذا.
- البيانات الأولية في `data/history.json` هي لقطة حقيقية واحدة بتاريخ 23 يونيو 2026 لتشغيل الصفحة فورًا قبل أول دورة تلقائية.
- توقيت GitHub Actions بتوقيت UTC (= توقيت السودان نفسه حاليًا، +0 فرق توقيت تقريبًا)، فلا حاجة لتحويل.
